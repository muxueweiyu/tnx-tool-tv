# 全局架构与防腐败定律 (00-global.md)

**作为 AI 辅助编写者，你必须绝对遵守以下代码架构规则，不得找任何借口绕过（由于追求快速实现而打破原则是被严格禁止的）：**

## 1. 严格分离与隔离层
项目分为 4 个强隔离区域，严格遵守依赖关系（单向向下依赖）：
- `src/commands`：只能负责解析终端输入，解析完毕必须调用 `services` 层。**【禁止包含如果/否则等业务分支】**。
- `src/services`：纯净无副作用的业务逻辑。**【禁止直接发起 HTTP 请求、禁止直接操作文件系统，必须调用 infrastructure】**。
- `src/infrastructure`：脏活区，任何外部 API（比如网络库）、系统 API 执行必须在这里。
- `src/core/models`：数据结构集中地。

## 2. 契约至上 (Data Contract Priority)
- **【严禁返回原生的 Dict / Record / Any】**。所有服务层的输入和输出，都必须是被 Type 注解过的数据模型（比如在 `models` 定义的 Interfaces）。
- 这解决了“加加加”导致的上下文浑浊问题：只要看 Model，就知道里面有什么字段。

## 3. 开闭原则 (Open-Closed Principle)
- 扩展系统不应通过修改现有代码实现。若需支持新行为、接入新平台，**必须新建一个实现相同接口的文件，而不是在原有函数里继续嵌套 `if...else...`**。
